## Bulma-native vs. extras

Most of the library maps directly onto [Bulma's documentation](https://bulma.io/documentation/):
each component page links to the Bulma element it wraps, and its props mirror
Bulma's modifier classes (`is-*`, `has-*`). If you already know Bulma, you
already know these components.

A few components are **extras** — net-new pieces with no Bulma equivalent,
built from Bulma primitives or filling gaps Bulma leaves to you:

- [ThemeToggle](/astro-bulma/components/themetoggle/) — light/dark mode switch
  built on Bulma 1.0's `data-theme` dark mode.
- [PageLayout](/astro-bulma/layouts/pagelayout/) — page header with title,
  actions, and notification slots.
- [SingleColumnLayout](/astro-bulma/layouts/singlecolumnlayout/) — centered
  single-column content wrapper.

They're marked with an <span class="sl-badge success">Extra</span> badge in
the sidebar.

## Installation

```sh
pnpm add astro-bulma bulma
```

Then import Bulma's CSS in your layout:

```astro
---
// src/layouts/Layout.astro
---
<html>
  <head>
    <link rel="stylesheet" href="/node_modules/bulma/css/bulma.min.css" />
  </head>
  <body><slot /></body>
</html>
```

Or import it in a global CSS file:

```css
@import "bulma/css/bulma.min.css";
```

## Usage

```astro
---
import { Button, Box, Notification } from "astro-bulma";
---

<Box>
  <Notification color="success" dismissible>
    Everything is working!
  </Notification>
  <Button color="primary" type="submit">Save</Button>
</Box>
```

## Individual imports

You can also import components directly from their path:

```astro
import Box from "astro-bulma/elements/Box.astro";
import TextInput from "astro-bulma/forms/TextInput.astro";
import Modal from "astro-bulma/components/Modal.astro";
```
