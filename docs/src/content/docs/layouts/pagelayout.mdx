The `PageLayout` component renders a Bulma level-based page header with a title and optional subtitle, then your page content below. It does not include an HTML shell — wrap it inside your app's base layout.

## Basic

<Preview>
  <PageLayout pageTitle="Users">
    <p>Page content goes here.</p>
  </PageLayout>
</Preview>

```astro
<PageLayout pageTitle="Users">
  <p>Page content goes here.</p>
</PageLayout>
```

## Props

| Prop           | Type                                                        | Default     | Description                                        |
|----------------|-------------------------------------------------------------|-------------|----------------------------------------------------|
| `pageTitle`    | `string`                                                    | —           | The page heading rendered in the level bar.        |
| `pageSubtitle` | `string`                                                    | —           | Optional subtitle rendered below the page heading. |
| `message`      | `string`                                                    | —           | If set, renders a `Notification` below the header. |
| `messageType`  | `"link" \| "primary" \| "info" \| "success" \| "warning" \| "danger"` | `"success"` | Color of the notification.            |

## Slots

| Slot      | Description                                                    |
|-----------|----------------------------------------------------------------|
| `before`  | Rendered above the level header bar.                           |
| `action`  | Rendered on the right side of the level header bar.            |
| default   | Main page content, rendered below the header and notification. |
| `modals`  | Rendered after the main content; intended for modal components.|
