The `Level` component renders Bulma's [level element](https://bulma.io/documentation/layout/level/). Put children in the `left` and `right` slots — each child should carry `class="level-item"` — or use the default slot alone for a centered level.

## Mobile

By default a level stacks vertically on mobile; `mobile` keeps it horizontal.

```astro
<Level mobile>…</Level>
```

## Props

| Prop     | Type      | Default | Description                            |
|----------|-----------|---------|----------------------------------------|
| `mobile` | `boolean` | —       | Keep the level horizontal on mobile.   |
| `class`  | `string`  | —       | Extra classes on the `nav.level`.      |
| `...`    | `*`       | —       | Any other HTML `nav` attributes.       |

### Slots

| Slot      | Description                                       |
|-----------|---------------------------------------------------|
| (default) | Centered `level-item` children.                   |
| `left`    | `level-item` children for the `level-left` side.  |
| `right`   | `level-item` children for the `level-right` side. |
