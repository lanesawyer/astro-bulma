The `Modal` component renders Bulma's [card-style modal](https://bulma.io/documentation/components/modal/). Toggle it by adding/removing `.is-active` on the modal's `id`. A built-in script handles closing via `data-modal-close` attributes and clicking the background.

## Basic modal

<Preview>
  <Button color="primary" onclick="document.getElementById('demo-modal').classList.add('is-active')">

  </Button>

  <Modal id="demo-modal" title="Example modal">
    <p>This is the modal body. Add any content here.</p>
  </Modal>

## With custom footer

The `footer` slot replaces the default Close button:

    Open confirm dialog

  <Modal id="confirm-modal" title="Confirm action">

    <Fragment slot="footer">
      <Button type="button" color="danger" data-modal-close>Delete</Button>
      <Button type="button" data-modal-close>Cancel</Button>

## Opening and closing

Toggle visibility by toggling `.is-active` on the element with the modal's `id`:

```js
// Open
document.getElementById("my-modal").classList.add("is-active");

// Close
document.getElementById("my-modal").classList.remove("is-active");
```

Any element with `data-modal-close` inside the modal (or the backdrop) will close it automatically.

## Props

| Prop    | Type     | Default | Description                                   |
|---------|----------|---------|-----------------------------------------------|
| `id`    | `string` | —       | **Required.** The modal element's `id`.      |
| `title` | `string` | —       | **Required.** Title shown in the modal header.|

## Slots

| Slot     | Description                                                                     |
|----------|---------------------------------------------------------------------------------|
| default  | Modal body content.                                                             |
| `footer` | Footer content. Defaults to a Close button with `data-modal-close`.            |
